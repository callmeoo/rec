# 快速部署指南

## 本地开发

### 启动开发服务器

```bash
npm run dev
```

访问地址：http://localhost:5173

### 在局域网内访问（手机测试）

1. 启动开发服务器
2. 查看终端输出的局域网地址，例如：
   ```
   ➜  Local:   http://localhost:5173/
   ➜  Network: http://192.168.1.100:5173/
   ```
3. 在手机浏览器中访问Network地址

## 部署到外网

### 方法1：使用Vercel（推荐，最简单）

1. 构建项目：
```bash
npm run build
```

2. 访问 https://vercel.com 并注册/登录

3. 点击 "Add New Project"

4. 选择导入方式：
   - 方式A：连接GitHub仓库（推荐）
   - 方式B：直接拖拽 `dist` 文件夹

5. 等待部署完成（约1-2分钟）

6. 获得外网访问链接，例如：
   ```
   https://your-project.vercel.app
   ```

### 方法2：使用Netlify

1. 构建项目：
```bash
npm run build
```

2. 访问 https://netlify.com 并注册/登录

3. 拖拽 `dist` 文件夹到Netlify页面

4. 获得外网访问链接，例如：
   ```
   https://your-project.netlify.app
   ```

### 方法3：使用自己的服务器

1. 构建项目：
```bash
npm run build
```

2. 将 `dist` 文件夹上传到服务器

3. 配置Nginx：

```nginx
server {
    listen 80;
    server_name your-domain.com;
    
    root /path/to/dist;
    index index.html;
    
    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

4. 重启Nginx：
```bash
sudo nginx -t
sudo systemctl restart nginx
```

## 响应式设计说明

本项目已支持响应式设计，可以在不同设备上自动适配：

- **移动端**（< 768px）：单列布局
- **平板端**（768px - 1024px）：双列布局
- **桌面端**（> 1024px）：三列或四列布局

### 测试响应式

1. 在浏览器中打开开发者工具（F12）
2. 点击设备工具栏图标（Ctrl+Shift+M）
3. 选择不同的设备尺寸进行测试

## 常见问题

### Q: 部署后页面刷新404

A: 需要配置服务器将所有请求重定向到index.html。Vercel和Netlify会自动处理。

### Q: 图片加载失败

A: 确保图片文件在 `public` 文件夹中，使用绝对路径 `/image.jpg`。

### Q: 移动端显示不正常

A: 检查viewport meta标签是否存在：
```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

## 性能优化建议

1. 图片优化：使用WebP格式，压缩图片大小
2. 启用Gzip压缩（Vercel/Netlify自动启用）
3. 使用CDN加速静态资源
4. 启用浏览器缓存
