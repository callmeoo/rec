# 友盟埋点集成说明

## 概述

推荐系统已集成友盟统计埋点，用于追踪用户行为和推荐效果。

**重要提示：** 友盟数据不是实时数据，有1天延迟。数据看板展示的是历史统计数据，不适合用于实时监控。

## 友盟SDK集成步骤

### 1. 安装友盟SDK
```bash
npm install umeng-web-analytics
```

### 2. 配置友盟AppKey
在 `src/utils/umengTracking.js` 中修改：
```javascript
const UMENG_APP_KEY = 'YOUR_UMENG_APP_KEY'; // 替换为实际的友盟AppKey
```

### 3. 初始化友盟
在 `src/main.jsx` 或 `src/App.jsx` 中调用：
```javascript
import { initUmeng } from './utils/umengTracking';

// 在应用启动时初始化
initUmeng();
```

### 4. 实现sendToUmeng函数
在 `src/utils/umengTracking.js` 中完善 `sendToUmeng` 函数，调用友盟SDK的实际API：
```javascript
const sendToUmeng = (eventData) => {
  // 调用友盟SDK
  if (window.aplus_queue) {
    window.aplus_queue.push({
      action: 'aplus.record',
      arguments: [eventData.event_name, 'CLK', eventData]
    });
  }
};
```

## 开发环境调试

当前在开发环境下，所有埋点数据会打印到浏览器控制台，格式如下：
```
[Umeng] 推荐模块曝光 {event_name: "recommendation_exposure", ...}
[Umeng] 车辆卡片点击 {event_name: "car_card_click", ...}
[Umeng] 页面浏览 {event_name: "page_view", ...}
```

## 注意事项

1. **用户ID获取**：当前使用模拟数据，实际使用时需要从用户登录状态中获取真实的用户ID
2. **候选集类型**：需要在推荐引擎中为每个车辆添加 `candidateSet` 字段
3. **推荐位位置**：需要在推荐列表中为每个车辆添加 `position` 字段（1-based索引）
4. **数据隐私**：确保埋点数据符合隐私政策和数据保护法规
5. **性能优化**：埋点调用应该是异步的，不应阻塞主线程

## 后续优化建议

1. **批量上报**：将多个埋点事件批量上报，减少网络请求
2. **离线缓存**：网络异常时将埋点数据缓存到本地，恢复后上报
3. **采样率控制**：对于高频事件可以设置采样率，减少数据量
4. **A/B测试**：集成A/B测试功能，对比不同推荐策略的效果
5. **实时监控**：添加实时数据监控和告警功能
