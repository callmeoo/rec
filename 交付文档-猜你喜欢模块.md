## 📊 推荐算法说明

### 评分机制

推荐算法基于多维度评分系统，总分由以下因素组成：

| 评分项 | 分值 | 说明 |
|--------|------|------|
| 品牌匹配 | +30 | 用户浏览过该品牌 |
| 价格相似 | +20/+10 | 与收藏车辆价格差<5万/10万 |
| 年份相似 | +15 | 与收藏车辆年份差≤1年 |
| 标签匹配 | +10/个 | 每个共同标签加10分 |
| 热度加成 | 0-20 | 基于浏览量（views/100） |
| 拍卖状态 | +15 | 正在拍卖中的车辆 |
| 低里程 | +10 | 里程<3万公里 |
| 新车 | +10 | 2022年及以后 |

### 推荐流程

```
用户行为数据 → 筛选条件过滤 → 评分计算 → 排序 → 生成推荐理由 → 展示
```

## 🎨 组件说明

### 1. RecommendationPage.jsx
**主页面组件**，负责整体布局和数据管理

**核心功能：**
- 管理推荐列表状态
- 处理用户交互（收藏、查看）
- 控制筛选面板显示
- 触发推荐刷新

**Props：** 无（顶层组件）

---

### 2. CarCard.jsx
**车辆卡片组件**，展示单个车辆信息

**Props：**
```javascript
{
  car: Object,           // 车辆数据对象
  onLike: Function,      // 收藏回调
  onView: Function,      // 查看回调
  isLiked: Boolean       // 是否已收藏
}
```

**车辆数据结构：**
```javascript
{
  id: Number,
  name: String,          // 车辆名称
  brand: String,         // 品牌
  year: Number,          // 年份
  price: Number,         // 价格（元）
  originalPrice: Number, // 原价（可选）
  mileage: Number,       // 里程（公里）
  location: String,      // 所在地
  views: Number,         // 浏览量
  image: String,         // 图片URL
  auctionStatus: String, // 拍卖状态（可选）
  tags: Array,           // 标签数组
  recommendReason: String // 推荐理由
}
```

---

### 3. FilterPanel.jsx
**筛选面板组件**，提供多维度筛选功能

**Props：**
```javascript
{
  filters: Object,       // 当前筛选条件
  setFilters: Function,  // 更新筛选条件
  onClose: Function      // 关闭面板回调
}
```

**筛选条件结构：**
```javascript
{
  priceRange: [min, max],    // 价格区间
  brands: Array,             // 品牌数组
  years: Array,              // 年份数组
  mileage: [min, max]        // 里程区间
}
```

---

### 4. recommendationEngine.js
**推荐算法引擎**，核心业务逻辑

**导出函数：**

#### `getRecommendations(userPreferences, filters)`
获取推荐车辆列表

**参数：**
```javascript
userPreferences: {
  viewedCars: Array,    // 浏览过的车辆ID
  likedCars: Array,     // 收藏的车辆ID
  searchHistory: Array  // 搜索历史
}

filters: {
  priceRange: [min, max],
  brands: Array,
  years: Array,
  mileage: [min, max]
}
```

**返回：** 排序后的推荐车辆数组

#### `getSimilarCars(carId, limit)`
获取相似车辆

**参数：**
- `carId`: 目标车辆ID
- `limit`: 返回数量（默认4）

**返回：** 相似车辆数组

## 🔧 配置说明

### 模拟数据
当前使用 `mockCars` 数组模拟车辆数据，包含12辆示例车辆。

**生产环境替换步骤：**

1. 在 `recommendationEngine.js` 中替换 `mockCars`：
```javascript
// 替换为API调用
const fetchCars = async () => {
  const response = await fetch('/api/cars');
  return response.json();
};
```

2. 更新 `RecommendationPage.jsx` 中的 `loadRecommendations`：
```javascript
const loadRecommendations = async () => {
  setLoading(true);
  try {
    const cars = await fetchCars(); // 调用真实API
    const recommendations = getRecommendations(userPreferences, filters);
    setRecommendations(recommendations);
  } catch (error) {
    console.error('加载推荐失败:', error);
  } finally {
    setLoading(false);
  }
};
```

### 用户偏好数据
当前用户偏好存储在组件状态中，生产环境建议：
- 使用 Redux/Zustand 等状态管理
- 持久化到 localStorage
- 同步到后端用户画像系统

## 🎯 API 接口建议

### 1. 获取推荐列表
```
GET /api/recommendations
Query Parameters:
  - userId: 用户ID
  - limit: 返回数量
  - filters: 筛选条件（JSON）

Response:
{
  "success": true,
  "data": [车辆数组],
  "total": 总数
}
```

### 2. 记录用户行为
```
POST /api/user/behavior
Body:
{
  "userId": "用户ID",
  "action": "view|like|search",
  "carId": "车辆ID",
  "timestamp": "时间戳"
}
```

### 3. 获取用户偏好
```
GET /api/user/preferences/:userId

Response:
{
  "viewedCars": [ID数组],
  "likedCars": [ID数组],
  "searchHistory": [搜索记录]
}
```
