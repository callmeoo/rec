# 为您推荐模块功能说明

## 概述

根据产品需求文档（PRD），本项目实现了完整的"为您推荐"模块，包括客户端推荐页面和管理后台验证功能。

## 已实现功能

### 1. 客户端 - 为您推荐页面

**访问路径**: 首页 → 猜你喜欢

**核心功能**:
- ✅ 个性化车辆推荐展示
- ✅ 车辆卡片信息完整展示
  - 品牌车系、车型、车牌
  - 价格信息（起拍价/当前价）
  - 车辆参数（车龄、里程、所在地）
  - 互动数据（浏览次数/人数、出价次数/人数）
- ✅ 促销标签优先级显示
  - 优先级1: 围观人数（浏览UV ≥ 100）
  - 优先级2: 多人意向（出价人数 > 3）
  - 优先级3: 快速周转（周转分 > 70）
- ✅ 车辆状态异常处理
  - 已下架车辆点击提示
  - 竞价结束车辆点击提示
- ✅ 筛选功能
  - 价格区间筛选
  - 品牌筛选
  - 车龄筛选
  - 里程筛选
- ✅ 换一批功能
- ✅ 收藏功能

**数据字段**:
```javascript
{
  id: 车辆ID,
  name: 车辆名称,
  brand: 品牌,
  year: 年份,
  price: 当前价格,
  originalPrice: 原价,
  mileage: 里程,
  location: 所在地,
  views: 浏览次数,
  viewUV: 浏览人数,
  bidCount: 出价次数,
  bidderCount: 出价人数,
  turnoverScore: 周转分,
  auctionStatus: 拍卖状态,
  status: 车辆状态 (active/ended/removed),
  recommendReason: 推荐理由
}
```

### 2. 管理后台 - 推荐算法验证（买家维度）

**访问路径**: 推荐管理 → 推荐算法验证

**核心功能**:
- ✅ 多方式买家搜索（支持买家ID、姓名、手机号）
- ✅ 买家基本信息展示（ID、姓名、电话、地区）
- ✅ 买家画像分析展示（新增功能）
  - 主打国别（如：日系 80%、德系 15%）
  - 主打品牌（如：大众 60%、丰田 20%）
  - 主打车系（如：大众迈腾 55%、丰田凯美瑞 30%）
  - 主打价位段（如：0-3万 35%、3-5万 28%）
  - 主打车龄（如：4-6年 61%、2-4年 25%）
  - 每个维度显示百分比和可视化进度条
- ✅ 固定展示30条推荐车源
- ✅ 完整的推荐数据列表
  - 排序（推荐位序号）
  - 推荐车源（品牌车系、车型、车牌）
  - 基本参数（国别、所在地、里程、车龄、车况评分）
  - 起拍价/状态（当前价格、倒计时）
  - 互动性（浏览次数/人数、出价次数/人数）
  - 推荐逻辑（推荐理由标签）
- ✅ 推荐逻辑标注
  - 画像匹配-主打车型（蓝色标签）
  - 候选车源-同车型高频成交（绿色标签）
  - 兜底-平台热销（灰色标签）
- ✅ 候选集标识（1/2/3）

**使用说明**:
1. 输入买家信息（支持三种方式）：
   - 买家ID（如：B001、B002、B003）
   - 买家姓名（如：张三、李四、王五）
   - 手机号（如：13812345678、13956781234）
2. 点击"查询"按钮
3. 查看买家画像分析（5个维度的百分比分布）
4. 查看该买家的推荐列表和推荐逻辑
3. 查看该买家的推荐列表和推荐逻辑

### 3. 管理后台 - 平台周转排行（系统维度）

**访问路径**: 推荐管理 → 平台周转排行

**核心功能**:
- ✅ 全平台统计数据概览
  - 车系总数
  - 最高周转分
  - 平均成交率
  - 平均耗时
- ✅ 车系维度排行榜
  - 按综合周转分降序排列
  - 前三名特殊标识（金、银、铜）
  - 展示车系名称、平均成交率、成交耗时中位数、综合评分
- ✅ 车型维度展开
  - 点击车系可展开查看具体车型
  - 展示车型名称、成交率、成交耗时、周转分、在拍数量
- ✅ 车辆详情穿透
  - 点击"查看详情"在新窗口打开车辆详情页
- ✅ 周转分颜色标识
  - ≥80分：绿色（优秀）
  - ≥70分：蓝色（良好）
  - ≥60分：橙色（一般）
  - <60分：红色（较差）

**周转分计算公式**:
```
周转分 = 成交率评分 × 0.7 + 耗时评分 × 0.3
```

## 技术实现

### 前端技术栈
- React 18
- Vite
- Tailwind CSS
- Lucide React (图标库)
- clsx (样式工具)

### 组件结构
```
src/
├── components/
│   ├── CarRecommendation/          # 客户端推荐模块
│   │   ├── RecommendationPage.jsx  # 推荐页面主组件
│   │   ├── CarCard.jsx             # 车辆卡片组件
│   │   ├── FilterPanel.jsx         # 筛选面板组件
│   │   └── recommendationEngine.js # 推荐引擎（模拟数据）
│   └── Admin/                      # 管理后台模块
│       ├── BuyerRecommendationVerify.jsx  # 推荐算法验证
│       └── TurnoverRanking.jsx            # 平台周转排行
```

### 数据模拟

当前使用模拟数据进行演示，实际生产环境需要对接后端API：

**需要对接的API**:
1. `GET /api/recommendations/:buyerId` - 获取推荐列表
2. `GET /api/vehicles/interaction-data` - 获取车辆实时数据
3. `GET /api/admin/recommendations/verify/:buyerId` - 管理后台推荐验证
4. `GET /api/admin/turnover-ranking` - 平台周转排行
