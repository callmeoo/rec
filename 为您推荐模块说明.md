# 为您推荐模块功能说明

## 概述

根据产品需求文档（PRD），本项目实现了完整的"为您推荐"模块，包括客户端推荐页面和管理后台验证功能。

## 已实现功能

### 1. 客户端 - 为您推荐页面

**访问路径**: 首页 → 猜你喜欢

**核心功能**:
- ✅ 个性化车辆推荐展示
- ✅ 车辆卡片信息完整展示
  - 品牌车系、车型、车牌
  - 价格信息（起拍价/当前价）
  - 车辆参数（车龄、里程、所在地）
  - 互动数据（浏览次数/人数、出价次数/人数）
- ✅ 促销标签优先级显示
  - 优先级1: 围观人数（浏览UV ≥ 100）
  - 优先级2: 多人意向（出价人数 > 3）
  - 优先级3: 快速周转（周转分 > 70）
- ✅ 车辆状态异常处理
  - 已下架车辆点击提示
  - 竞价结束车辆点击提示
- ✅ 筛选功能
  - 价格区间筛选
  - 品牌筛选
  - 车龄筛选
  - 里程筛选
- ✅ 收藏功能

**数据字段**:
```javascript
{
  id: 车辆ID,
  name: 车辆名称,
  brand: 品牌,
  year: 年份,
  price: 当前价格,
  originalPrice: 原价,
  mileage: 里程,
  location: 所在地,
  views: 浏览次数,
  viewUV: 浏览人数,
  bidCount: 出价次数,
  bidderCount: 出价人数,
  turnoverScore: 周转分,
  auctionStatus: 拍卖状态,
  status: 车辆状态 (active/ended/removed),
  recommendReason: 推荐理由
}
```

**核心功能**:

- ✅ 买家基本信息展示（ID、姓名、电话、地区）

  - 主打车系（如：大众迈腾 55%、丰田凯美瑞 30%）
  - 主打价位段（如：0-3万 35%、3-5万 28%）
  - 主打车龄（如：4-6年 61%、2-4年 25%）
  - 每个维度显示百分比和可视化进度条
- ✅ 固定展示30条推荐车源
- ✅ 完整的推荐数据列表

  - 推荐车源（品牌车系、车型、车牌）
  - 基本参数（国别、所在地、里程、车龄、车况评分）

  - 画像匹配-主打车型（蓝色标签）

  - 兜底-平台热销（灰色标签）
- ✅ 候选集标识（1/2/3）

**使用说明**:
1. 输入买家信息（支持三种方式）：
   - 买家ID（如：B001、B002、B003）
   - 买家姓名（如：张三、李四、王五）
   - 手机号（如：13812345678、13956781234）
2. 点击"查询"按钮
3. 查看买家画像分析（5个维度的百分比分布）
4. 查看该买家的推荐列表和推荐逻辑
3. 查看该买家的推荐列表和推荐逻辑

### 3. 管理后台 - 平台周转排行（系统维度）

**访问路径**: 推荐管理 → 平台周转排行

**核心功能**:
- ✅ 全平台统计数据概览
  - 车系总数
  - 最高周转分
  - 平均成交率
  - 平均耗时
- ✅ 车系维度排行榜
  - 按综合周转分降序排列
  - 前三名特殊标识（金、银、铜）
  - 展示车系名称、平均成交率、成交耗时中位数、综合评分
- ✅ 车型维度展开
  - 点击车系可展开查看具体车型
  - 展示车型名称、成交率、成交耗时、周转分、在拍数量
- ✅ 车辆详情穿透
  - 点击"查看详情"在新窗口打开车辆详情页
- ✅ 周转分颜色标识
  - ≥80分：绿色（优秀）
  - ≥70分：蓝色（良好）
  - ≥60分：橙色（一般）
  - <60分：红色（较差）

**周转分计算公式**:
```
周转分 = 成交率评分 × 0.7 + 耗时评分 × 0.3
```

## 推荐策略

### 1. 数据筛选和过滤

- 车辆状态：预展中、拍卖中
- 过滤车况评级D和火烧车、泡水车（含"水泡"标签）
- 过滤已登录账号已出价车辆

### 2. 推荐数据

#### 2.1 前端展示

前端展示全平台上拍车辆的15%（按照每日上架不足200台，为双排推荐**固定展示16台车，可配**）

#### 2.2 候选集

##### 候选集 1

**Step 1 — 强匹配（优先）**

- 匹配维度：主打车系 + 主打价格段 + 主打车龄（三维全匹配）
  - 例如买家标签为"大众迈腾 + 3-5万 + 4-6年"，则严格按取该交集下的车源，若满18台则停止向下寻找。
- 若结果 > 目标数量 → 直接使用，反之进入Step 2

**Step 2 — 双维宽泛匹配（解决候选偏差大）**

- 匹配条件：【主打车系】+【价位区差：金额×10%】+【车龄区差：±1年】。
- 动作：将符合该条件的车辆补充进列表（需排除第1层级已有的车，按vid去重）。
- 触发条件：前段1结果 < 16 台。
- 价格容差逻辑：在原价位段的基准值上，允许上下浮动10%的绝对金额，价格不可为负；当最低价格段为"0-3万"时，向下容差计算可出现负数（例如0万的-10%应该是0），价格容差不可为负。
  - 举例：如果买家标签是3-5万，允许推荐起拍价在2.7万（3万-10%）至5.5万（5万+10%）之间的同车系，这样能扩充车源，让候选集能接近2.98万的车源也进入匹配范围。
- 车龄容差逻辑：在买家主打车龄标签的上下限上，各放宽1年。
  - 举例：若买家主打车龄为4-6年，"容差"扩为3-7年。若买家主打车龄为2年以内（即0-2年）：向下不可为负，向上放宽年，容差扩为0-3年。若买家主打车龄为10年以上：向下放宽，容差扩为9年以上。
- 若结果 > 目标数量 → 直接使用，反之进入Step 3

**Step 3 — 放开车系（一级扩展）**

- 匹配维度：主打价格段 + 主打车龄（去掉车系限制）
- 若结果 > 目标数量 → 使用，反之进入Step 4

**Step 4 — 兜底（二级扩展）**

- 匹配维度：无约束，从候选集2（平台置顶TOP50）中补充剩余缺口
- 若候选集2仍不足，则从候选集3（全站兜底）补至目标数量
- 去重机制：在经历阶段1到阶段4的漏斗过滤时，拉取的数据集可能会有重合，需做去重处理，避免同一台车在候选展示两次。

##### 候选集 2（平台置顶）

- 全站置顶分TOP 50车系，同车系占比 < 40%；候选集内的车辆需要差异大于60；

##### 候选集 3（全站兜底）

- 若以上候选结果集都不足，则余车辆推荐位用"全站热销兜底"。
- 全站热销：在拍车辆按照出价次数从高到低排序，出价次数一致，按余价格结束时间升序排列；若拍卖结束时间和出价次数都是相同，则采随机排序打散。
- 在全局过滤规则外，候选集3排除出价次数为0的车辆

#### 2.3 排序规则

- 同一个候选集：按【拍卖结束时间(升序)】，拍卖结束时间相同，按出价次数(降序)；若拍卖结束时间和出价次数相同，随机排序。
- 打散策略：前10位中，同一品牌连续 ≤3 台，同一车系连续 ≤3 台。

#### 2.4 异常处理

- 为保持系统高可用，当推荐服务出现故障，接口超时未返回，则直接调用**缓存**中的上一次计算结果。
- 在该取缓存列表后，将状态不符（已拍卖结束的）车辆从缓存中过滤，若数量不足，则用候选集3（全站热销兜底）进行补充。
- **三个候选集集合并后仍不足16台时，保持实际数量展示，不强制补满**

#### 2.5 用户未登录情况下的推荐策略

- 取：候选集2 & 候选集3 数据

### 3. 数据流

#### 3.1 推荐生成流程

- 买家访问 → 提取买家ID → 查询60天出价历史 → 生成买家画像
- 生成候选集1（画像匹配）→ 生成候选集2（平台置顶）→ 生成候选集3（兜底）
- 合并候选集 → 应用打散策略 → 返回推荐列表

#### 3.2 实时更新流程

- 前端定时器（30秒）→ 读取车辆互动数据 → 更新卡片展示
- 检测车辆状态变化 → 移除失效车辆 → 补充新推荐

#### 3.3 管理后台运营流程

- 运营/产品人员选择买家 → 调用推荐引擎 → 生成30条推荐 → 标注推荐逻辑
