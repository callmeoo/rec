## 主要修改

### 1. CarCard 组件 (src/components/CarRecommendation/CarCard.jsx)

#### 修改内容：

**促销标签计算逻辑 (getPromotionTag 函数)**
- 实现了三级优先级的促销标签显示
- 添加了缺失字段的默认值处理（使用 `|| 0`）

```javascript
const getPromotionTag = () => {
  const viewUV = car.viewUV || 0;
  const bidderCount = car.bidderCount || 0;
  const turnoverScore = car.turnoverScore || 0;
  
  // 优先级1: 围观人数（浏览UV ≥ 100）
  if (viewUV >= 100) {
    return { text: '围观人数', color: 'bg-red-500' };
  }
  
  // 优先级2: 多人意向（出价人数 > 3）
  if (bidderCount > 3) {
    return { text: '多人意向', color: 'bg-orange-500' };
  }
  
  // 优先级3: 快速周转（周转分 > 70）
  if (turnoverScore > 70) {
    return { text: '周转快', color: 'bg-green-500' };
  }
  
  return null;
};
```

**促销标签渲染逻辑**
- 在互动数据区域显示促销标签
- 添加了出价人数显示（当 bidderCount > 0 时）

```jsx
{car.viewUV && (
  <div className="flex gap-3 mb-3 text-xs text-slate-500 border-t border-slate-100 pt-3">
    <span>{car.viewUV}人围观</span>
    {car.bidderCount > 0 && <span>{car.bidderCount}人出价</span>}
    {promotionTag && (
      <span className={`px-2 py-0.5 ${promotionTag.color} text-white rounded`}>
        {promotionTag.text}
      </span>
    )}
  </div>
)}
```

## 功能特性

### 促销标签优先级

1. **优先级1: 围观人数**（红色标签）
   - 条件：浏览UV ≥ 100
   - 样式：红色背景 (bg-red-500)
   - 文本：白色

2. **优先级2: 多人意向**（橙色标签）
   - 条件：浏览UV < 100 且 出价人数 > 3
   - 样式：橙色背景 (bg-orange-500)
   - 文本：白色

3. **优先级3: 快速周转**（绿色标签）
   - 条件：浏览UV < 100 且 出价人数 ≤ 3 且 周转分 > 70
   - 样式：绿色背景 (bg-green-500)
   - 文本：白色

### 车辆状态异常处理（已确认）

- 已下架车辆（status='removed'）：点击时显示"该车辆已下架，请刷新页面"
- 竞价结束车辆（status='ended'）：点击时显示"该车辆竞价已结束，请刷新页面"
- 正常车辆（status='active'）：正常跳转到详情页

### 数据完整性

- 所有车辆数据包含必要字段
- 字段值在合理范围内
- 缺失字段自动使用默认值 0

## 注意事项

1. **边界值处理**：
   - viewUV = 100 会显示"围观人数"（包含边界值）
   - bidderCount = 3 不会显示"多人意向"（不包含边界值）
   - turnoverScore = 70 不会显示"周转快"（不包含边界值）

2. **缺失字段处理**：
   - 如果车辆数据缺少 viewUV、bidderCount 或 turnoverScore 字段
   - 系统会自动将其视为 0
   - 不会导致程序崩溃

3. **单一标签原则**：
   - 每个车辆卡片最多显示一个促销标签
   - 按优先级从高到低判断
   - 满足高优先级条件后不再检查低优先级
