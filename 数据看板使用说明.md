# 数据看板使用说明

## 访问路径

**推荐管理 > 数据看板**

## 功能概述

数据看板用于查看推荐系统的核心指标和效果分析，帮助运营人员了解推荐模块的表现。

## 重要提示

⚠️ **数据特性：**
- 数据来源于友盟统计平台
- **非实时数据**，通常有5-30分钟延迟
- 适合用于历史数据分析和趋势观察
- 不适合用于实时监控

## 功能模块

### 1. 核心指标卡片

展示推荐系统的5个核心指标：

| 指标 | 说明 |
|------|------|
| 总曝光量 | 推荐模块在选定时间范围内的总曝光次数 |
| 总点击量 | 用户点击推荐车辆的总次数 |
| 点击率(CTR) | 点击量/曝光量，反映推荐内容的吸引力 |
| 出价率 | 出价次数/点击次数，反映推荐质量 |
| 平均出价 | 用户平均出价金额（万元） |

每个指标卡片还会显示与上周的对比数据（↑ 增长 / ↓ 下降）。

### 2. 日期范围选择

支持3个时间维度：
- **近7天**：查看最近一周的数据趋势
- **近30天**：查看最近一个月的数据趋势
- **近90天**：查看最近三个月的数据趋势

切换日期范围后，所有数据会自动更新。

### 3. 趋势分析图表

以柱状图形式展示每日的数据趋势：
- **蓝色柱**：曝光量
- **绿色柱**：点击量
- **橙色柱**：出价量

**交互功能：**
- 鼠标悬停在柱状图上可查看当天的详细数据
- 横轴显示日期（月-日格式）
- 纵轴高度按各指标的最大值自动缩放

### 4. 候选集命中分布

展示推荐车辆来源的分布情况：
- **画像匹配**：基于买家画像匹配的车辆占比
- **平台周转**：基于平台周转分推荐的车辆占比
- **全站兜底**：兜底推荐的车辆占比

以进度条形式直观展示各候选集的占比。

### 5. 设备分布

展示PC端和移动端的数据分布：
- 曝光量
- 点击量
- 点击率(CTR)

帮助了解不同设备上的用户行为差异。

### 6. 推荐位效果分析

以表格形式展示不同推荐位的效果：

| 推荐位 | 指标 |
|--------|------|
| 1-3位 | 曝光量、点击量、点击率、出价率 |
| 4-6位 | 曝光量、点击量、点击率、出价率 |
| 7-10位 | 曝光量、点击量、点击率、出价率 |
| 10位后 | 曝光量、点击量、点击率、出价率 |

**颜色标识：**
- 绿色：表现良好（CTR≥7%，出价率≥12%）
- 橙色：表现一般（低于阈值）

## 操作按钮

### 刷新按钮
点击刷新按钮可重新加载数据（实际使用时会从友盟服务器获取最新数据）。

### 导出报表按钮
点击可导出当前数据为Excel报表（功能开发中）。

## 数据更新机制

### 开发环境
- 展示模拟数据
- 数据根据选择的日期范围动态生成
- 每次刷新会生成新的随机数据

### 生产环境
- 数据从友盟统计平台获取
- 更新频率：5-30分钟延迟
- 数据按日期维度聚合

## 埋点事件

数据看板展示的数据来自以下埋点事件：

1. **推荐模块曝光** (`recommendation_exposure`)
   - 触发时机：推荐列表加载完成
   - 记录：用户ID、设备类型、车辆ID列表、曝光数量

2. **车辆卡片点击** (`car_card_click`)
   - 触发时机：用户点击车辆卡片
   - 记录：用户ID、车辆ID、品牌、型号、推荐位、候选集、设备类型

3. **页面浏览** (`page_view`)
   - 触发时机：页面加载/刷新
   - 记录：页面名称、设备类型、登录状态

4. **出价行为** (`bid_action`)
   - 触发时机：用户出价
   - 记录：用户ID、车辆ID、出价金额、推荐位、候选集

## 使用场景

### 1. 日常监控
- 每日查看核心指标的变化趋势
- 关注点击率和出价率的波动
- 对比不同时间段的数据表现

### 2. 效果评估
- 评估推荐算法的整体效果
- 分析不同候选集的贡献度
- 对比不同推荐位的转化效果

### 3. 优化决策
- 根据设备分布优化移动端体验
- 根据推荐位效果调整展示策略
- 根据候选集分布优化推荐算法

### 4. 数据报告
- 导出数据用于周报/月报
- 向管理层汇报推荐系统表现
- 与其他业务数据进行关联分析

## 注意事项

1. **数据延迟**：友盟数据有5-30分钟延迟，不要用于实时决策
2. **数据准确性**：确保埋点代码正确部署，否则数据可能不准确
3. **时区问题**：注意数据的时区设置，确保与业务时区一致
4. **采样率**：高流量场景下可能需要设置采样率，注意数据是否经过采样
5. **隐私合规**：确保数据收集和使用符合隐私政策和法律法规

## 常见问题

### Q1: 为什么数据看板显示的数据和实际感觉不一致？
A: 友盟数据有延迟，建议等待30分钟后再查看。同时检查埋点代码是否正确部署。

### Q2: 如何查看更详细的数据？
A: 可以登录友盟统计后台查看更详细的数据和自定义报表。

### Q3: 数据看板支持自定义时间范围吗？
A: 当前仅支持7天、30天、90天三个固定范围。如需自定义，请联系技术团队。

### Q4: 导出的报表包含哪些数据？
A: 导出功能开发中，计划包含所有看板展示的数据，支持Excel格式。

### Q5: 数据看板的数据可以用于A/B测试吗？
A: 可以，但需要在埋点中添加实验分组标识，并在友盟后台配置相应的分析维度。
